### Architecture Recommendations

1. **强制 ALS 内传 userId，单例无状态化**：sidecar 所有 PriorityMerger / queue / merger 类禁止 instance 级 uid/row/chain 字段；统一通过 `runWithUser(uid, ...)` ALS 透传，并在 CI gate 扩展 regex 拦截 module-level Array/Object/Map 任何 uid-tagged 容器。直接消除 finding #2 / #29 / #43 一整类。

2. **single-source-of-truth shared helpers + Zod schemas**：
   - 存储上限公式抽 `@ogamex/shared/storage_cap.ts`（修 #5）。
   - section_settings 白名单 + 值类型抽 Zod schema in `@shared/section_settings.ts`，Next route / sidecar sectionSettingsWrite / billing.ts auto-unpause 全走同一 schema（修 #25）。
   - 三处 PG RMW 改单语句 `||` jsonb merge（修 #26）。

3. **billing webhook 二阶段 idempotency + 服务端 plan re-derive**：
   - `recordPaymentEvent` 改 `pending`→`processed` 双态，handler 成功才 mark processed；Stripe/PayPal 同形态一并修（#6/#7）。
   - 所有客户端可控 plan 透传必须服务端从 PayPal/Stripe 真值 re-derive（#3）。
   - gift-code 铸只在 checkout/onApprove 路径或基于 `subscriptions.first_activated_at` 而非事件类型（#8）。

4. **网络/secret 收敛 + fail-loud**：
   - sidecar bind 127.0.0.1；deploy.sh 默认 secret 改空 + die；NODE_ENV=production 时若 RESEND_API_KEY/SMTP_HOST 全空 → die；删除 PAYPAL_SKIP_WEBHOOK_VERIFY 或 gate 到非 prod（#1/#15/#11/#18/#20）。
   - Bearer 全部走 `crypto.timingSafeEqual`（#30）。
   - 接入最小化 rate-limit（IP+email LRU + free-code per-account 阈值）（#32）。

5. **schema/retention 显式化**：
   - drizzle 立即生成 baseline migration，prod 禁 `db:push`；ogame_world_state.json 顶层加 `schema_version`（#12）。
   - ogame_events / payment_events / verification_tokens 三表分别上 cron retention + 缺失索引（#4/#13/#27）。
   - subscriptions 改 set-null cascade + payment_events 加 userId 列（#14/#27）。
